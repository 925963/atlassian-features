## Description

Bitbucket Cloud is introducing merge queues to help teams maintain stable, deployable branches in high-traffic repositories. Instead of merging pull requests directly, Bitbucket adds them to a merge queue that sequences pull requests to be merged into a branch one by one, ensuring each change is compatible with the latest code. This keeps the target branch green and reduces failed builds by validating combinations of changes before they land. To get started with this change: Go to Repository set
