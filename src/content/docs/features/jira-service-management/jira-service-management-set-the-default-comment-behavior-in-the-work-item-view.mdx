## Description

We're introducing the ability for admins to set the default behavior for comments on the work item view (in addition to transition screens, which has already been released). Admins can now configure the default comment behavior at the instance level, ensuring that the setting aligns with their organization's communication preferences. As part of this change, admins can choose whether comments default to 'Internal note' or 'Reply to customer' in both the transition screen and the work item view,
