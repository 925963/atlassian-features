## Description

In Jira Service Management, Assets automation now includes a new Attribute value changed trigger. You can use it to run rules when specific object attributes change, instead of only reacting to object-level events like create, update, or delete. This gives admins more precise control over automation and can reduce manual updates and workaround rules. To get started with this change: In Jira Service Management, go to a project that uses Assets and open Project settings.
